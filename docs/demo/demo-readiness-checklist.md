# Demo Readiness Checklist

## Platform

- [ ] Backend running at `http://localhost:4000/api/v1`
- [ ] Swagger opens at `http://localhost:4000/api/docs`
- [ ] PostgreSQL is running and migrations are applied
- [ ] Database seed completed successfully
- [ ] Customer app opens
- [ ] Rider app opens
- [ ] Vendor dashboard opens at `http://localhost:3000`
- [ ] Admin portal opens at `http://localhost:3001`

## Demo Data

- [ ] Demo customer can log in
- [ ] Demo vendor can log in and is active/open
- [ ] Demo rider can log in, is approved, and can go online
- [ ] Demo admin can log in
- [ ] Demo vendor has Jollof Rice and Chicken Suya products
- [ ] Demo customer has a Home address
- [ ] `KARIGOFIRST` exists and is active

## Demo Flows

- [ ] Mock payment initiation and verification work
- [ ] Vendor receives and progresses paid order
- [ ] Dispatch board loads available rider and ready order
- [ ] Rider accepts, progresses, and completes with OTP
- [ ] Notifications/activity feeds update
- [ ] Customer support ticket can be created
- [ ] Admin can respond and add an internal note
- [ ] Dashboard, reports, settlements, and earnings load

## Presentation

- [ ] KariGO logo is visible and not distorted on all four surfaces
- [ ] API URLs point to the correct demo backend
- [ ] Browser/mobile screens are pre-opened in journey order
- [ ] Known issues and mock-provider limitations have been reviewed
- [ ] Real credentials and production/customer data are not used
- [ ] Presenter knows the recovery plan if a step fails
