# Internal Demo Checklist

## Before

- [ ] Backend running; database migrated and seeded
- [ ] Customer/rider apps and vendor/admin dashboards running
- [ ] Demo accounts and one clean order journey ready
- [ ] Internet/test devices/screenshare stable; backup screenshots available
- [ ] Known issues, mock-provider limits, and recovery steps reviewed

## During

- [ ] Customer order/payment/tracking works
- [ ] Vendor fulfilment and admin dispatch work
- [ ] Rider delivery/OTP completion works
- [ ] Support, reports, notifications, settlements, and earnings load

## After

- [ ] Feedback captured; bugs entered in staging register
- [ ] Management decision recorded
- [ ] Owners and due dates assigned
