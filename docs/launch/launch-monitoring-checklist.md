# Launch Monitoring Checklist

## Technical

- [ ] API uptime, latency, slow endpoints, and error rate
- [ ] Database connectivity, query failures, storage, and connection usage
- [ ] Login failures, invalid tokens, and role-guard denials
- [ ] Payment verification/webhook failures and duplicates
- [ ] Customer/rider app crashes and dashboard errors

## Operations

- [ ] Orders created, paid, vendor accepted/rejected, and ready for pickup
- [ ] Rider assigned, completed, failed, and delayed deliveries
- [ ] Refund requests and approvals
- [ ] Open/critical support tickets
- [ ] Average preparation and delivery time when accurate timestamps are available

## Finance

- [ ] GMV and delivery fees
- [ ] Refund amount
- [ ] Pending/paid vendor settlements
- [ ] Pending/paid rider earnings
- [ ] Daily reconciliation differences

## Customer

- [ ] Registrations, first orders, repeat orders, and promo usage
- [ ] Complaints by category and severity
- [ ] Notification delivery/read state
- [ ] Customer-impacting incidents and unresolved refunds
