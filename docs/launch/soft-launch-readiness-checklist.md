# Soft Launch Readiness Checklist

## Technical Readiness

- [ ] Backend deployed and production database connected.
- [ ] Health endpoint and Swagger visibility decision verified.
- [ ] Customer app and rider app installed on pilot devices.
- [ ] Vendor dashboard and admin portal accessible over HTTPS.
- [x] Mock payment, order statuses, vendor, rider, support, promo, and notifications tested.
- [ ] Real-provider decision and production security review completed.

## Operations Readiness

- [ ] Super admin, operations admin, dispatch officer, support officer, and finance officer ready.
- [ ] Vendor, rider, and customer test accounts ready.
- [ ] Initial vendors and riders onboarded and verified.
- [ ] Escalation contact list and shift ownership approved.
- [ ] Daily operations/finance report process agreed.

## Business Readiness

- [ ] Launch areas, vendor list, rider list, and approved promo confirmed.
- [ ] Support number and escalation hours approved.
- [ ] Website content, launch visuals, announcements, and onboarding forms ready.
- [ ] Terms, privacy, refund, vendor, and rider policies reviewed.

## Launch Risks

- [ ] Payment failure and refund process rehearsed.
- [ ] Vendor rejection, rider no-show, unreachable customer, and late-delivery process rehearsed.
- [ ] App downtime and admin access-recovery process rehearsed.
- [ ] Database backup/restore process tested.
- [ ] Daily performance and incident review scheduled.
