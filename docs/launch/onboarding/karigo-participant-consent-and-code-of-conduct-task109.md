# KariGO Participant Consent And Code Of Conduct - Task 109

Date prepared: 2026-07-13

## Purpose

Provide pilot participants with clear expectations before they receive controlled early
access.

## Participant Acknowledgement

By joining the KariGO controlled early access pilot, participants acknowledge that:

- KariGO is not fully publicly launched.
- Access is limited to approved pilot users in Kano.
- Some services are in pilot, test mode or readiness-only.
- Feedback may be requested after app usage, order activity or support interaction.
- Access may be paused, suspended or revoked if pilot rules are breached.

## Code Of Conduct

Participants must:

- use only the app/dashboard links shared by KariGO;
- keep login details private;
- avoid sharing screenshots containing private information;
- report issues through approved support channels;
- follow vendor, customer, Delivery Captain or admin instructions for their assigned role;
- avoid abusive, unsafe or misleading conduct during pilot activity.

Participants must not:

- share passwords, OTPs, delivery codes, APK links or private access links publicly;
- ask another participant for passwords, OTPs, delivery codes or card details;
- attempt to access another role's dashboard/app;
- record private customer/vendor/Delivery Captain data in public channels;
- claim live rides, utilities, wallet withdrawals, payouts or Pharmacy marketplace are active.

## Role-Specific Reminders

| Role | Reminder |
| --- | --- |
| Customer | Share delivery code only after receiving the order |
| Vendor | Accept only available orders and update preparation status accurately |
| Delivery Captain | Use delivery code only at handoff; Ride Captain remains readiness-only |
| Operations Admin | Handle participant data only for approved pilot operations |
