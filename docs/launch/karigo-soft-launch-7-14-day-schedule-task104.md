# KariGO Soft Launch 7-14 Day Schedule - Task 104

Date prepared: 2026-07-13

## Recommended Schedule

The first Kano pilot should run for 7 to 14 days, starting with tightly supervised
low-volume testing before any expansion.

| Day | Focus | Activities | Decision |
| --- | --- | --- | --- |
| Day -2 | Setup | Confirm roster, install apps, verify accounts, brief vendors and Delivery Captains | Ready / Not ready |
| Day -1 | Dry run | Backend health, Admin/Vendor domains, Customer/Captain login, mock payment smoke test | Ready / Fix first |
| Day 1 | First supervised order | Run one end-to-end order with all role owners present | Continue / Pause |
| Day 2 | Limited order batch | 2 to 5 controlled orders across selected vendors | Continue / Adjust |
| Day 3 | Support and dispatch focus | Test support tickets, reassignment if safe, vendor timing | Continue / Adjust |
| Day 4 | Settlement/earning visibility | Confirm completed order records, vendor settlement, Captain earning visibility | Continue / Adjust |
| Day 5 | SME Services review | Submit and review safe SME Services request if in pilot scope | Continue / Adjust |
| Day 6 | Reliability review | Repeat core flows and update issue register | Continue / Pause |
| Day 7 | Midpoint decision | Review metrics, P0/P1 issues, support load and operations capacity | Continue / Extend / Stop |
| Days 8-13 | Optional extension | Controlled volume increase only if stable | Continue / Stop |
| Day 14 | Closeout | Final report, decision and follow-up fix plan | Expand / Continue / Pause |

## Daily Standup Agenda

1. Yesterday's order count and completion count.
2. Vendor response issues.
3. Delivery Captain response issues.
4. Customer/support issues.
5. Payment/test-mode issues.
6. Backend/app/domain issues.
7. Open P0/P1/P2 issues.
8. Today's order and support plan.
9. Decision: continue, continue with conditions, pause or stop.

## Daily Closeout Questions

- Did any order fail because of a platform issue?
- Did any customer, vendor or Delivery Captain need manual workaround?
- Did support resolve issues within the agreed response window?
- Did any sensitive data appear in screenshots, logs or documents?
- Did any live provider activate unexpectedly?
- Are we safe to continue tomorrow?

## Expansion Rule

Do not expand beyond the approved pilot group unless:

- no P0/P1 issue is open;
- daily operations report is stable;
- support load is manageable;
- vendors and Delivery Captains can handle the next batch;
- management approves expansion in writing.
