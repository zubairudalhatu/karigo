# Public Launch Readiness Checklist

## Product

- [ ] Customer app stable
- [ ] Rider app stable
- [ ] Vendor dashboard stable
- [ ] Admin portal stable
- [ ] Order flow stable
- [ ] Payment flow stable
- [ ] Dispatch flow stable
- [ ] Support flow stable
- [ ] Refund flow stable
- [ ] Notification flow stable
- [ ] Critical/high pilot bugs closed and regression evidence attached

## Operations

- [ ] Vendors signed and onboarded
- [ ] Riders verified and onboarded
- [ ] Dispatch team ready
- [ ] Support team ready
- [ ] Finance process ready
- [ ] Daily reporting process ready
- [ ] Incident escalation ready
- [ ] Vendor and rider training completed
- [ ] Capacity and rollback limits approved for opening zones/hours

## Provider Readiness

- [ ] Payment provider has live approval
- [ ] SMS OTP provider has live approval
- [ ] Email provider ready
- [ ] WhatsApp provider approved, if used
- [ ] Push notification provider ready, if used
- [ ] Webhooks verified
- [ ] Live credentials secured, scoped, rotatable, and absent from Git

## Infrastructure

- [ ] Production hosting, database, backups/restore, monitoring, HTTPS, CORS, rate limits, and session hardening approved

## Legal And Policy

- [ ] Terms of Service approved
- [ ] Privacy Policy approved
- [ ] Refund Policy approved
- [ ] Vendor Agreement approved
- [ ] Rider Agreement approved
- [ ] Data-protection compliance reviewed

## Marketing And Distribution

- [ ] Website ready
- [ ] App-store listing ready
- [ ] Launch visuals ready
- [ ] Social-media plan ready
- [ ] Vendor launch campaign ready
- [ ] Rider recruitment campaign ready
- [ ] Customer launch offer approved with abuse controls
- [ ] Public announcement approved

## Decision

- [ ] Ready for public launch
- [ ] Ready for extended controlled pilot only
- [x] Not currently ready: existing provider/infrastructure/legal/physical-device gates remain open
